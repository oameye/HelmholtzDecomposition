# Changelog

## v2.0.1

Bug fix.

- **`AutocalculateSingleFij` rejected polynomial terms with rational-in-parameter coefficients** (e.g. `x / (1 + a^2)`). Cause: `PolynomialQ[S, xiWithoutxk]` with `xiWithoutxk == {}` enters Mathematica's auto-detect mode and refuses any expression with a rational sub-piece, even when the rational dependence is on parameters not in the coordinate list. Fix: test `PolynomialQ[S, xvec]` instead — checks polynomiality in the spatial coordinates, with parameters treated as constants. Two regression tests added in `api.wlt`.

## v2.0.0

Major rewrite: one entry point, queryable result object.

### Breaking changes

- The seven v1 functions (`HelmholtzGradient`, `HelmholtzRotational`, `HelmholtzPotential`, `HelmholtzVectorPotential`, `HelmholtzAssociation`, `PrintHelmholtz`) are removed. Use `HelmholtzDecomposition[f, xvec][prop]` instead.
- `HelmholtzDecomposition[f, xvec]` no longer returns `{g, r}`. It returns a `HelmholtzDecomposition[<|...|>]` object. Use `List @@ hd` to recover the legacy tuple shape.
- The `Simplify` option is renamed to `SimplificationFunction` to avoid shadowing the built-in symbol.
- `SetVerbose` / `GetVerbose` (global state) are replaced by a per-call `Verbose -> True | False` option.
- The `VectorPotential` slot is renamed `AntisymmetricPotential` (it's a 2-form in general; the 3D vector `A` is now the separate `VectorPotentialA` property).

### Additions

- Properties accessible via `hd["name"]`: `"Gradient"`, `"Rotational"`, `"Potential"`, `"AntisymmetricPotential"`, `"PotentialMatrix"`, `"Residual"`, `"Verify"`, `"VectorPotentialA"` (3D only), `"Field"`, `"Coordinates"`, `"Properties"`, `"Association"`.
- `"Verify"` returns `<|"GradientCurlFree", "RotationalDivergenceFree", "ResidualZero"|>` — precomputed, always uses `FullSimplify` regardless of the user's `SimplificationFunction`.
- `Caching -> True | False` option (default True). `ClearHelmholtzCache[]` empties the cache, returns the eviction count.
- `TimeConstraint -> Infinity | N` option, forwarded to the simplifier as a circuit breaker for long simplifications.
- Coordinate auto-detection: `HelmholtzDecomposition[f]` extracts `xvec` from the symbols in `f`.
- `MakeBoxes` rule produces a collapsible summary box in notebook display.
- `List @@ hd → {g, r}`, `Normal[hd] → Association`, `Length[hd] → field dimension`.
- Documentation paclet pages under `Documentation/English/{Guides, ReferencePages/Symbols, Tutorials}`, generated by `build/build_docs.wls`.
- Failure-mode tests covering: length mismatch, ambiguous coordinate auto-detection, undecomposable atoms (`Throw`/`Catch`'d to `$Failed` rather than `Abort`'d), unknown property, 3D-only property in lower dimension, non-boolean `Verbose`/`Caching`, bad `TimeConstraint`, malformed wrapper to `MakeBoxes`.
- `SyntaxInformation` set for IDE auto-complete.

### Internal

- Single-shot `hdCompute` builds the full Association in one pass; all properties are stored, not lazy.
- Memoization keyed on the full `(f, xvec, simp, asm, tc)` tuple — no Hash collisions.
- Private throw tag isolates internal `$Failed` propagation from user-level `Catch`.
- Property-based test assertions in `examples.wlt` (using `Simplify[diff] === 0`) survive `FullSimplify` canonical-form drift across Mathematica versions.
